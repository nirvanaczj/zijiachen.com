Prompt: To create a better search experience for users.

Idea:
With my exploration on Universal Sentence Encoders, I figured that different length of sentence/phrases can be encoded into same length arrays. Therefore, I got an idea to develop a basic recommendation system for Stady based on user's course name input and tutor courses history, and furthermore, compare and return tutors who took similar courses as user's input.

Technical:
I used bert-as-service for sentence encoding and reduced mean pooling for pooling high dimension arrays. I encoded all tutors' courses into same length arrays and save them in a .txt file updating every night. In this way, new searchs won't need the search engine to re-encode all tutors' courses. Moreover, I utlized Cosine Similarity instead of Euclidean Distance to compare array because it turns out to perform better while testing. After all, I deployed such model to google cloud platform's virtual machine with Flask. I also developed frontend search page for stadyapp.com

Utility:
Stady Semantic Search engine allows students to search for tutors related to their current classes. For example, by searching “Statistical Inference”, the search engine can return tutors who have previously taken courses on probability.
